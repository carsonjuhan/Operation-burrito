## Description

Add subtle page transition animations and micro-interactions to improve perceived performance and visual polish. Animations must be CSS-only where possible, lightweight, and respect the `prefers-reduced-motion` media query so users who opt out of motion see no animations.

---

## Acceptance Criteria

- [ ] Page content fades in on route change via a reusable wrapper component
- [ ] List items animate in with a staggered reveal (CSS animation-delay)
- [ ] Toggle actions (pack/unpack, purchase/unpurchase) have smooth transitions
- [ ] All animations respect `prefers-reduced-motion` media query (disabled when set to `reduce`)
- [ ] No jank or layout shift caused by animations
- [ ] `npm run build` and `npx vitest run` pass with no regressions

---

## Technical Context

### Current State
- No page transition animations exist
- Tailwind config already has a `shake` keyframe/animation for the contraction timer
- `globals.css` defines component classes in `@layer components`
- Layout renders `{children}` directly inside `<main>` with no transition wrapper
- Toggle actions (hospital bag pack/unpack, item purchase) use instant state changes

### Implementation Plan

1. **`tailwind.config.ts`** -- Add keyframes and animation utilities
   - `fadeIn`: opacity 0 to 1 over 300ms
   - `slideUp`: translateY(8px) + opacity 0 to translateY(0) + opacity 1
   - `staggerItem`: same as slideUp but designed for use with CSS custom property `--stagger-index` for delay

2. **`src/app/globals.css`** -- Add animation utility classes
   - `.animate-page-in`: fade-in animation for page content
   - `.animate-stagger-item`: stagger animation for list items using `--stagger-index`
   - `.animate-toggle`: smooth transition for toggle state changes
   - `@media (prefers-reduced-motion: reduce)` block that disables all custom animations

3. **`src/components/PageTransition.tsx`** -- Create lightweight wrapper component
   - Client component that wraps page content with `animate-page-in` class
   - Simple div wrapper, no animation library needed

4. **Apply to pages** -- Add animation classes to:
   - Dashboard (`src/app/page.tsx`): wrap in PageTransition, stagger stat cards
   - Items page (`src/app/items/page.tsx`): stagger list items
   - Hospital bag page (`src/app/hospital-bag/page.tsx`): stagger bag items, animate pack toggle
   - All other pages: wrap in PageTransition for fade-in
