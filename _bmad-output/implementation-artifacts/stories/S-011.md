## Description

The current auto-sync (debounced push after every store mutation) silently swallows all errors. If the GitHub API is unreachable, the PAT has expired, or the Gist ID is invalid, the user has no indication that their data is not being backed up. This story adds error surfacing, failure counting, auto-pause after repeated failures, and differentiated error messages.

---

## Acceptance Criteria

- [ ] Failed auto-sync attempts show a toast notification (non-intrusive)
- [ ] Sidebar sync indicator changes to a warning state on failure
- [ ] After 3 consecutive failures, auto-sync pauses and shows a persistent warning in the sidebar
- [ ] User can manually retry sync from the sidebar warning or dismiss it
- [ ] Network errors vs. authentication errors show different messages
- [ ] Successful sync after failure clears the warning state and resets the failure counter
- [ ] Unit tests cover error classification, failure counting, and state transitions
- [ ] `npm run build` and `npx vitest run` pass with no regressions

## Technical Context

### Current State
- `src/hooks/useStore.ts` -- `triggerAutoSync()` catches errors silently (`catch { /* silent */ }`)
- `src/lib/gistSync.ts` -- `pushToGist()` throws generic `Error` objects with status codes in messages
- `src/components/Sidebar.tsx` -- Shows GitHub connected/not-connected status but no error state
- No toast/notification system exists in the codebase yet

### Implementation Plan

1. **`src/lib/syncErrorHandler.ts`** -- New module for sync error logic
   - `SyncErrorType` enum: `"network"`, `"auth"`, `"unknown"`
   - `classifySyncError(error: Error): SyncErrorType` -- inspects error message/status to determine type
   - `getSyncErrorMessage(type: SyncErrorType): string` -- returns user-friendly message per type
   - `SyncFailureTracker` class or functions to track consecutive failure count, paused state

2. **`src/hooks/useAutoSyncStatus.ts`** -- New hook for auto-sync error state
   - Exports `AutoSyncStatus` type: `{ state, consecutiveFailures, lastError, errorType, isPaused }`
   - `useAutoSyncStatus()` hook providing state and actions: `recordFailure()`, `recordSuccess()`, `retry()`, `dismiss()`
   - Pauses auto-sync after 3 consecutive failures
   - Provides toast-triggering callback on failure

3. **`src/hooks/useStore.ts`** -- Modifications
   - Import and use `useAutoSyncStatus` to track sync outcomes
   - In `triggerAutoSync`, call `recordFailure()` or `recordSuccess()` based on result
   - Skip auto-sync when status is paused
   - Expose `autoSyncStatus` and `retrySyncNow()` from the hook return

4. **`src/components/Sidebar.tsx`** -- Modifications
   - Show warning indicator when auto-sync has failed
   - Show persistent warning with retry/dismiss when paused (3+ failures)
   - Use amber/red colors for warning states

5. **`src/components/Toast.tsx`** -- New component (lightweight)
   - Simple toast container + toast component with success/error/warning/info variants
   - Fixed position bottom-right, auto-dismiss after 4 seconds
   - Toast context for app-wide use

6. **Tests** -- `src/lib/__tests__/syncErrorHandler.test.ts`
   - Error classification (network vs auth vs unknown)
   - Failure tracking (count, pause at 3, reset on success)
   - Error message differentiation
