## Description

Validate data integrity when pulling from GitHub Gist. Ensure the pulled JSON conforms to the `AppStore` schema before overwriting local data. Reject or warn on malformed data. Save a pre-pull snapshot for recovery.

Currently, `pullFromGist()` in `gistSync.ts` does a raw `JSON.parse()` and casts the result directly to `AppStore` with no validation. If the remote data is malformed, incomplete, or from an older schema version, the local state silently corrupts. This story adds a validation layer, default-filling for backward compatibility, and a recovery snapshot mechanism.

---

## Acceptance Criteria

- [ ] Pulled data is validated against expected shape (all required arrays present, correct field types)
- [ ] Missing fields are filled with defaults (backward-compatible migration)
- [ ] Completely invalid JSON shows an error message instead of silently corrupting local state
- [ ] A pre-pull snapshot is saved to a separate localStorage key for recovery
- [ ] User can restore from the snapshot if the pull results in data corruption
- [ ] Unit tests verify validation logic for valid, partial, and invalid data
- [ ] `npm run build` and `npx vitest run` pass with no regressions

## Technical Context

### Current State
- `src/lib/gistSync.ts` -- `loadGist()` does `JSON.parse(raw) as AppStore` with no validation
- `src/hooks/useStore.ts` -- `loadStore()` already does some default-filling when loading from localStorage, but the sync pull path bypasses this
- `src/app/settings/page.tsx` -- `handlePull()` calls `pullFromGist()` and passes result directly to `loadFromExternal()`
- `src/types/index.ts` -- defines `AppStore` and all entity interfaces

### Implementation Plan

1. **`src/lib/syncValidation.ts`** -- New validation module
   - `validateAppStore(data: unknown): { valid: boolean; store: AppStore | null; errors: string[] }` -- validates shape and fills defaults
   - Checks: top-level is an object, each array field is actually an array, birthPlan is an object with expected nested shape
   - For each missing top-level field, fills with the default value (empty array or default birth plan)
   - Returns errors array describing what was wrong for completely invalid data
   - `SNAPSHOT_KEY = "ob-pre-pull-snapshot"` -- localStorage key for recovery snapshot
   - `savePrePullSnapshot(store: AppStore): void` -- saves current store before pull
   - `getPrePullSnapshot(): AppStore | null` -- retrieves saved snapshot
   - `clearPrePullSnapshot(): void` -- removes snapshot after successful use

2. **`src/lib/gistSync.ts`** -- Modifications
   - Import and use `validateAppStore` in `loadGist()` to validate parsed data
   - Throw descriptive error if validation fails completely
   - Return the validated + default-filled store

3. **`src/app/settings/page.tsx`** -- Modifications
   - Before pulling, save a pre-pull snapshot via `savePrePullSnapshot()`
   - On pull error, show a "Restore previous data" button
   - On successful pull with filled defaults, show an info message

4. **Tests** -- `src/lib/__tests__/syncValidation.test.ts`
   - Valid complete AppStore passes validation
   - Partial AppStore with missing arrays gets defaults filled
   - Completely invalid data (string, number, null) returns errors
   - Missing birthPlan gets default filled
   - Extra unknown fields are preserved (forward compatibility)
   - Snapshot save/load/clear works correctly
