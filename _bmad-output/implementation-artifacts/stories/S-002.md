## Description

Write comprehensive unit tests for the five critical utility modules: `csvImporter.ts`, `receiptParser.ts`, `inventoryMatching.ts`, `importParsers.ts`, and `gistSync.ts`. These modules contain the core data transformation and sync logic of the app and currently have zero test coverage.

The testing framework (Vitest + React Testing Library) was set up in S-001. This story uses pure unit tests (no DOM rendering needed) for all modules except where mocking `fetch` or `localStorage` is required.

---

## Acceptance Criteria

- [ ] `csvImporter.ts` tests cover: valid CSV parsing, quoted fields with escaped quotes, column auto-mapping from various header synonyms, Google Sheets URL conversion, normalizeCategory, normalizePriority, normalizePurchased, normalizePrice, mapRowToItem, and edge cases (empty rows, missing headers, empty input)
- [ ] `receiptParser.ts` tests cover: text-to-item parsing with prices, line filtering (totals, tax, dates, phone numbers excluded), price extraction at end of line, items without prices auto-deselected, duplicate filtering, cleanName stripping quantity prefixes
- [ ] `inventoryMatching.ts` tests cover: exact match, normalized match via manual mappings, unmatched items returning false, getSuggestedMatches returning high/medium confidence results
- [ ] `importParsers.ts` tests cover: ICS event parsing (with TZID, UTC, date-only), vCard contact parsing (FN, N fallback, TEL, EMAIL, ORG/TITLE), EML header extraction (From with name+email, subject, body with HTML stripping), extractDateTimeFromText (ISO, named month, numeric, time with am/pm)
- [ ] `gistSync.ts` tests cover: PAT/Gist ID persistence helpers (get/set/clear using mocked localStorage), push/pull API calls with mocked fetch, error handling for failed API calls
- [ ] All tests pass via `npm test`
- [ ] No changes to production source code are required

## Technical Context

### Current State
- Vitest is configured with `jsdom` environment, `globals: true`, and `@/*` path alias (from S-001)
- Test setup file at `src/test/setup.ts` imports `@testing-library/jest-dom/vitest`
- Test scripts: `npm test` (single run), `npm run test:watch` (watch mode)
- All utility modules are pure TypeScript with no React dependencies
- `gistSync.ts` uses `fetch` and `localStorage` which need mocking
- `receiptParser.ts` uses `crypto.randomUUID()` which needs mocking in jsdom

### Architecture Decisions
- **Pure unit tests** -- no component rendering needed, just import functions and assert
- **Mock fetch globally** via `vi.fn()` for gistSync tests
- **Mock localStorage** via `vi.stubGlobal` or jsdom's built-in implementation
- **Mock crypto.randomUUID** for deterministic receipt parser test output

### Relevant Source Files
- `src/lib/csvImporter.ts` -- CSV parsing, column mapping, value normalization
- `src/lib/receiptParser.ts` -- Receipt text parsing, price extraction
- `src/lib/inventoryMatching.ts` -- Registry-to-checklist matching
- `src/lib/importParsers.ts` -- ICS, vCard, EML parsers
- `src/lib/gistSync.ts` -- GitHub Gist CRUD and config helpers
- `src/types/index.ts` -- TypeScript interfaces used by all modules

## Implementation Tasks

### Task 1: Write csvImporter tests (~45 min)

Create `src/lib/__tests__/csvImporter.test.ts`:
- `parseCsv`: basic CSV, quoted fields, escaped quotes, empty input, empty rows skipped
- `autoMapColumns`: maps common synonyms (item, product, price, etc.)
- `normalizeCategory`: maps known categories, defaults to "Other"
- `normalizePriority`: maps synonyms, defaults to "Nice to Have"
- `normalizePurchased`: truthy values return true, others false
- `normalizePrice`: strips `$`, commas, spaces; returns undefined for invalid
- `mapRowToItem`: maps a full row, returns null for missing name
- `toGoogleSheetsCsvUrl`: extracts spreadsheet ID and gid

### Task 2: Write receiptParser tests (~30 min)

Create `src/lib/__tests__/receiptParser.test.ts`:
- `parseReceiptText`: parses lines with trailing prices, filters skip patterns, deduplicates, auto-deselects items without price
- Mock `crypto.randomUUID` for deterministic IDs

### Task 3: Write inventoryMatching tests (~20 min)

Create `src/lib/__tests__/inventoryMatching.test.ts`:
- `findChecklistMatch`: exact match, manual mapping match, no match
- `getSuggestedMatches`: returns high-confidence manual matches, medium-confidence keyword matches, limits to 5

### Task 4: Write importParsers tests (~40 min)

Create `src/lib/__tests__/importParsers.test.ts`:
- `parseIcs`: single event, multiple events, TZID dates, UTC dates, date-only, line folding, organizer CN
- `parseVCard`: single contact, multiple contacts, FN vs N fallback, ORG/TITLE in notes
- `parseEml`: standard email, name+email from, subject, body HTML stripping, quoted-printable decoding
- `extractDateTimeFromText`: ISO date, named month, numeric date, time with am/pm

### Task 5: Write gistSync tests (~30 min)

Create `src/lib/__tests__/gistSync.test.ts`:
- PAT helpers: `getPAT`, `setPAT`, `getGistId`, `setGistId`, `clearGistConfig`
- `verifyPAT`: mocked fetch success/failure
- `createGist`, `updateGist`, `loadGist`: mocked fetch with correct URL/method/body assertions
- `pushToGist`: creates new gist when no ID, updates existing when ID present
- `pullFromGist`: throws when no gist ID, calls loadGist when ID present

### Task 6: Verify all tests pass (~10 min)

- Run `npm test` and confirm all tests pass
- Run `npm run build` to confirm no build breakage

## Notes

- **`crypto.randomUUID`** is used in `receiptParser.ts`. In jsdom it may not be available. Mock it with `vi.stubGlobal('crypto', { randomUUID: vi.fn(() => 'test-uuid') })` or use `vi.spyOn`.
- **`fetch`** is used in `gistSync.ts` and `csvImporter.ts` (fetchGoogleSheetCsv). Mock with `vi.stubGlobal('fetch', vi.fn())`.
- **`localStorage`** is available in jsdom by default, so gistSync helper tests should work without extra mocking.
- The `ocrImageFile` and `extractPdfText` async functions in receiptParser.ts are NOT tested here -- they require heavy dependencies (tesseract.js, pdfjs-dist). Only `parseReceiptText` (the pure text parser) is tested.
- The `fetchGoogleSheetCsv` async function is NOT tested here since it is a thin fetch wrapper. Only `toGoogleSheetsCsvUrl` (the URL transformer) is tested.
