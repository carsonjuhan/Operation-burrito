# S-026: Shareable Read-Only Birth Plan Link

**Epic:** E-006 (Collaboration and Sharing)
**Sprint:** 5
**Story Points:** 5
**Status:** In Progress
**Priority:** P2

## Description

Generate a shareable URL that allows others (doulas, family, hospital staff) to view the birth plan without editing access. Since this is a static site with no backend, the birth plan data is compressed and encoded into the URL fragment (hash). A dedicated `/birth-plan/view` route reads the hash, decodes it, and renders a clean read-only view. Link revocation is achieved by regenerating the link (which changes the encoded data).

## Acceptance Criteria

- [ ] "Share Birth Plan" button on the Birth Plan page opens a share dialog
- [ ] Clicking share encodes the current birth plan as base64 JSON in the URL hash fragment
- [ ] Link is copyable via a "Copy Link" button with confirmation feedback
- [ ] `/birth-plan/view` route reads the hash, decodes the birth plan, and renders a read-only view
- [ ] Read-only view is clean and styled without navigation sidebar, editing controls, or tabs
- [ ] Read-only view clearly indicates it is read-only (banner or header note)
- [ ] Shared view works without authentication (no AuthGate)
- [ ] Empty or invalid hash shows a friendly error message
- [ ] Sharer can revoke access by regenerating the link (editing their plan changes the hash)
- [ ] Works entirely client-side with no external dependencies

## Tasks

1. **Create birth plan encoding utilities** -- `encodeBirthPlan(plan)` compresses BirthPlan to base64 string; `decodeBirthPlan(hash)` reverses it with error handling
2. **Add Share button to birth plan page** -- Button opens a modal/dialog showing the generated link with a Copy button
3. **Create `/birth-plan/view` route** -- New page that reads hash from URL, decodes it, and renders read-only birth plan
4. **Create read-only birth plan renderer** -- Reuse the existing `BirthPlanPrintView` display components adapted for screen viewing (not just print)
5. **Handle edge cases** -- Invalid/missing hash, empty birth plan, and URL length limits
6. **Bypass AuthGate for view route** -- Ensure the shared link works without password

## Technical Notes

- Use `btoa(JSON.stringify(plan))` for encoding and `JSON.parse(atob(hash))` for decoding
- URL fragment (hash) is never sent to the server, preserving privacy
- The view page reads `window.location.hash` on the client side
- Birth plan data is typically 2-5KB JSON which encodes to ~3-7KB base64 -- well within URL limits
- The view route uses its own layout without sidebar/AuthGate, OR the existing AuthGate is bypassed for this route
- Reuse the print view display components (PrintSection, PrintField, PrintCheck, PrintValue) for the read-only screen view
