# S-025: Bundle pdfjs-dist Worker Locally

**Epic:** E-005 (Import, Export, and Print Enhancements)
**Sprint:** 4
**Story Points:** 2
**Status:** In Progress
**Priority:** P2

## Description

The PDF.js worker is currently loaded from the unpkg CDN (`https://unpkg.com/pdfjs-dist@.../build/pdf.worker.min.mjs`), creating an external runtime dependency. Bundle it locally so PDF parsing works fully offline after the initial app load.

## Acceptance Criteria

- [ ] PDF.js worker file is copied to the `public/` directory
- [ ] `receiptParser.ts` references the local worker path instead of unpkg URL
- [ ] Local worker path accounts for the production `basePath` (`/Operation-burrito`)
- [ ] PDF parsing works without internet access (after initial app load)
- [ ] No increase in the main JavaScript bundle size (worker is a separate file)
- [ ] Build succeeds with the local worker reference

## Tasks

1. Copy `node_modules/pdfjs-dist/build/pdf.worker.min.mjs` to `public/pdf.worker.min.mjs`
2. Update `extractPdfText()` in `receiptParser.ts` to reference the local worker path
3. Account for `basePath` in the worker URL (production uses `/Operation-burrito` prefix)
4. Verify build succeeds

## Technical Notes

- The basePath pattern used elsewhere: `const basePath = process.env.NODE_ENV === 'production' ? '/Operation-burrito' : '';`
- Worker file is ~220KB minified, served as a separate static file from `public/`
- The worker file should be copied manually (not via a build script) since it changes infrequently
- When pdfjs-dist is upgraded, the worker file in public/ must be updated to match

## Dependencies

- None

## Files to Modify

- `public/pdf.worker.min.mjs` -- Copy of the worker file from node_modules
- `src/lib/receiptParser.ts` -- Update workerSrc to reference local path with basePath
