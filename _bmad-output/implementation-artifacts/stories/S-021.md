# S-021: Birth Plan PDF Export

**Epic:** E-005 (Import, Export, and Print Enhancements)
**Sprint:** 4
**Story Points:** 8
**Status:** In Progress
**Priority:** P2

## Description

Generate a clean, printable PDF of the completed birth plan for distribution to hospital staff, doulas, and support people. Uses the browser's built-in `window.print()` with `@media print` CSS rather than a heavy library like jsPDF -- this is lighter, works offline, and produces better results for this use case.

## Acceptance Criteria

- [ ] "Export as PDF" button on the Birth Plan page triggers the browser print dialog
- [ ] Print layout includes all completed birth plan sections in a clear, readable format
- [ ] Sections with no data are omitted or marked as "Not specified"
- [ ] Print header includes patient name, due date, and generation date
- [ ] Print layout is formatted for standard letter/A4 paper
- [ ] Navigation, buttons, and non-essential UI are hidden in print view
- [ ] All four tab panels are visible when printing (not just the active tab)
- [ ] Form inputs render as plain text values in print view
- [ ] Print footer notes the document was generated by Operation Burrito
- [ ] Works entirely client-side with no external dependencies

## Tasks

1. **Create BirthPlanPrintView component** -- A dedicated print-only component that renders all birth plan sections in a clean, structured layout with proper headings and value display
2. **Enhance @media print CSS** -- Extend existing print styles in globals.css for letter/A4 formatting, proper margins, page break control, and footer
3. **Handle empty sections** -- Show "Not specified" for empty text fields and omit fully empty sections
4. **Print header** -- Display patient name (legal + preferred), due date, and "Generated on [date]"
5. **Print footer** -- Add "Generated by Operation Burrito" footer on each printed page
6. **Hide interactive elements** -- Ensure all buttons, tabs, checkboxes (as inputs), and navigation are hidden; show values as static text
7. **Update birth plan page** -- Integrate the print view component alongside the existing interactive form

## Technical Notes

- Use `window.print()` which allows the user to save as PDF via their browser's print dialog
- The existing `@media print` rules in globals.css already hide the sidebar and `.print-hide` elements, and show all `.birth-plan-panel` sections
- The existing print header in birth-plan/page.tsx provides a starting point but needs enhancement
- Checkboxes should render as checkmark/cross text in print view rather than HTML checkbox inputs
- Use `@page` CSS rule for margins and paper size
- Use CSS `display: none` for screen-only elements and `display: block` for print-only elements via Tailwind's `hidden print:block` pattern

## Dependencies
