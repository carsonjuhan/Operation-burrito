## Description

Improve the receipt import flow with better progress indication, error messaging, and the ability to review/edit parsed items in a table before importing. Currently the modal shows a spinner during OCR/PDF processing with no progress feedback, error messages are generic, and while items can be edited inline they lack a structured table layout. This story adds a progress bar, restructures the review step as an editable table with select/deselect checkboxes, and provides clearer error messages for unsupported file types and parsing failures.

---

## Acceptance Criteria

- [ ] Progress bar shows OCR/PDF parsing progress (determinate where possible, indeterminate otherwise)
- [ ] "Loading scanner..." state shown while tesseract.js / pdfjs-dist libraries load
- [ ] Parsed items are displayed in a structured editable table before import (header row with Name, Price columns)
- [ ] User can correct item names and prices inline before confirming import
- [ ] User can deselect individual items they do not want to import via checkbox
- [ ] Clear, specific error messages for unsupported file types (listing accepted formats)
- [ ] Clear error messages for parsing failures (e.g., unreadable image, empty PDF)
- [ ] `npm run build` and `npx vitest run` pass with no regressions

## Technical Context

### Current State
- `src/components/ReceiptImportModal.tsx` -- Existing modal with upload, spinner loading state, and inline item review
- `src/lib/receiptParser.ts` -- `parseReceiptFile()`, `ocrImageFile()`, `extractPdfText()` functions; tesseract.js worker logger callback is suppressed
- The OCR worker in `ocrImageFile()` accepts a `logger` callback that receives progress events with `{ status, progress }` shape
- PDF extraction iterates pages sequentially, allowing page-level progress

### Implementation Plan

1. **`src/lib/receiptParser.ts`** -- Modifications
   - Add `onProgress?: (pct: number, message: string) => void` callback parameter to `ocrImageFile()` and `extractPdfText()`
   - In `ocrImageFile()`: use the tesseract.js `logger` callback to forward `progress` (0-1) values as percentage
   - In `extractPdfText()`: report progress based on `currentPage / totalPages`
   - Update `parseReceiptFile()` to accept and forward the progress callback

2. **`src/components/ReceiptImportModal.tsx`** -- Modifications
   - Replace the spinner-only loading state with a progress bar (0-100%)
   - Show phase labels: "Loading scanner...", "Recognizing text...", "Extracting text from PDF..."
   - Restructure the review step as a table with header (checkbox, Name, Price)
   - Add Select All / Deselect All controls in the table header
   - Make error messages more specific:
     - Unsupported file type: list accepted formats explicitly
     - Empty parse result: suggest retrying with a clearer photo or different file
     - Library load failure: suggest refreshing the page
   - Use `size="lg"` on the Modal for the review step to give the table more room
