## Description

Implement a lightweight "show more" pagination pattern for the baby items list to avoid rendering hundreds of DOM nodes at once on low-end mobile devices. When a category group exceeds 50 items, only the first 50 are rendered initially, with a "Show more" button to load additional batches. This avoids adding heavy virtualization libraries (react-window, @tanstack/virtual) while still providing smooth performance for lists that may grow to a few hundred items. CSS `content-visibility: auto` is also applied to category sections for native browser-level rendering optimization.

## Technical Context

### Current State
- `src/app/items/page.tsx` -- Items page renders all items in each category group at once via `items.map()` inside collapsed sections
- Items are grouped by `ItemCategory` and displayed inside collapsible `<div>` sections
- The `MergedItemRow` component renders each individual item row
- Typical item counts: ~100 checklist items + custom tracked items, grouped across 9 categories

### Implementation Plan

1. **`src/hooks/useShowMore.ts`** -- New custom hook for pagination state
   - `useShowMore(totalCount: number, pageSize?: number)` returns `{ visibleCount, showMore, hasMore, remaining, reset }`
   - Tracks how many items to show per group
   - `showMore()` increments visible count by `pageSize` (default 50)
   - `reset()` resets to initial page size (called when filters change)
   - Pure logic, easily testable

2. **`src/app/items/page.tsx`** -- Modifications
   - Import and use `useShowMore` for each category group
   - Slice `items` array to `visibleCount` before mapping to `MergedItemRow`
   - Render "Show more" button when `hasMore` is true
   - Add `content-visibility: auto` and `contain-intrinsic-size` CSS to category card containers
   - Reset visible counts when filter state changes

3. **Tests** -- `src/hooks/__tests__/useShowMore.test.ts`
   - Initial visible count equals page size or total if smaller
   - `showMore()` increments by page size
   - `hasMore` is false when all items are visible
   - `remaining` returns correct count
   - `reset()` returns to initial state
   - Custom page size works correctly
