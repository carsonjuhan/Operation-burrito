## Description

Implement a hand-written service worker that caches the static shell (HTML, JS, CSS, icons) and enables the app to work offline after the initial visit. The service worker uses a cache-first strategy for static assets and a network-first strategy for GitHub Gist API calls. Offline sync operations are queued for replay when connectivity returns.

The service worker is a plain JavaScript file placed in `public/sw.js` so it is served at the site root after Next.js static export. Registration is handled by a React hook (`useServiceWorker`) that is only active in production builds (respecting `basePath`).

## Acceptance Criteria

- [ ] Service worker is registered on first visit in production
- [ ] All static assets (HTML, JS, CSS, icons) are cached on install
- [ ] App loads and is fully functional offline after initial cache
- [ ] Static assets use cache-first strategy (fast loads, offline support)
- [ ] Gist API calls use network-first strategy (fresh data when online, cached fallback)
- [ ] Sync operations attempted while offline are queued in localStorage
- [ ] Cache is updated when a new version is deployed (versioned cache name, old caches pruned)
- [ ] Service worker does not register in development mode
- [ ] Unit tests cover the registration hook logic

## Technical Context

### Current State
- The app is a **static export** deployed to GitHub Pages at `/Operation-burrito/`
- `next.config.js` sets `basePath` and `assetPrefix` to `/Operation-burrito` in production
- All data is stored in localStorage; GitHub Gist sync is optional
- No service worker or PWA support exists yet
- `public/` directory does not currently exist (icons and manifest are planned for S-019)

### Architecture Decisions
- **ADR: Hand-written SW over next-pwa/Workbox** -- The caching requirements are straightforward (static shell + one API domain). A hand-written ~100-line service worker is simpler to understand and debug than pulling in Workbox or next-pwa, which add significant bundle complexity for marginal benefit at this scale.
- **ADR: Cache versioning via constant** -- A `CACHE_VERSION` constant in sw.js is bumped on deploy. The activate event prunes old versioned caches. This is the standard stale-while-revalidate pattern for static sites.
- **ADR: Offline sync queue in localStorage** -- Failed sync operations are serialized to localStorage so they survive page refreshes. S-020 will add the UI indicator and automatic replay.

### Relevant Files
- `next.config.js` -- basePath/assetPrefix configuration
- `src/app/layout.tsx` -- root layout where SW registration component could be added
- `src/lib/gistSync.ts` -- Gist API calls that need network-first caching
- `public/sw.js` -- the service worker file (to be created)

## Implementation Tasks

### Task 1: Create the service worker file (est. 30 min)

Create `public/sw.js` with:
- `CACHE_VERSION` constant for cache busting
- `install` event: pre-cache the static shell (basePath-aware URLs)
- `activate` event: delete old versioned caches
- `fetch` event handler with two strategies:
  - **Cache-first** for same-origin static assets (HTML, JS, CSS, images, fonts)
  - **Network-first** for `api.github.com` requests (Gist API)
- Offline queue: when a Gist API request fails due to network, store the request details in a message to the client for localStorage persistence

### Task 2: Create the registration hook (est. 20 min)

Create `src/hooks/useServiceWorker.ts`:
- Check `process.env.NODE_ENV === 'production'` before registering
- Register `sw.js` with the correct scope (basePath-aware)
- Handle the `updatefound` event to detect new versions
- Listen for `controllerchange` to notify user of updates
- Export the hook for use in a client component

### Task 3: Create the SW registration component (est. 10 min)

Create `src/components/ServiceWorkerRegistration.tsx`:
- `"use client"` component that calls `useServiceWorker()`
- Renders nothing (returns null) -- purely side-effect
- Added to `layout.tsx` inside the body

### Task 4: Write tests for the registration hook (est. 30 min)

Create `src/hooks/__tests__/useServiceWorker.test.ts`:
- Mock `navigator.serviceWorker` and its `register` method
- Test that registration is skipped in development
- Test that registration calls `navigator.serviceWorker.register` with correct scope
- Test update detection via `updatefound` event

### Task 5: Verify build and tests (est. 10 min)

- Run `npx vitest run` -- all tests pass
- Run `npm run build` -- static export succeeds with sw.js in output
- Verify sw.js is present in the `out/` directory
