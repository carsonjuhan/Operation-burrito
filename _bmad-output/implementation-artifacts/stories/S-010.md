## Description

Allow users to import a previously exported JSON file to restore their data, as an alternative to GitHub Gist sync. The import reuses the `validateAppStore()` validation from S-007 to ensure data integrity before replacing local state.

Currently, the only way to restore data is via GitHub Gist pull. Users who do not use GitHub (or whose Gist was lost) have no recovery path. This story also adds the complementary JSON export button (S-009) which is referenced in the sprint but missing from the settings page UI.

---

## Acceptance Criteria

- [ ] "Import Data" button on the Settings page opens a file picker for `.json` files
- [ ] Imported data is validated against the AppStore schema (reuses S-007 `validateAppStore`)
- [ ] Confirmation dialog warns that import will replace all local data
- [ ] On success, the store is updated and a success message is shown
- [ ] On validation failure, an error message describes what was wrong
- [ ] "Export Data" button triggers a JSON file download with timestamped filename
- [ ] Unit tests verify the import validation and file handling logic
- [ ] `npm run build` and `npx vitest run` pass with no regressions

## Technical Context

### Current State
- `src/lib/syncValidation.ts` -- `validateAppStore()` already validates and default-fills AppStore data (from S-007)
- `src/app/settings/page.tsx` -- Settings page has GitHub sync and storage sections but no import/export buttons
- `src/hooks/useStore.ts` -- `loadFromExternal()` replaces the store with new data and persists to localStorage

### Implementation Plan

1. **`src/lib/dataImportExport.ts`** -- New module for import/export logic
   - `exportStoreAsJSON(store: AppStore): void` -- triggers a timestamped JSON file download
   - `parseImportFile(file: File): Promise<ValidationResult>` -- reads file, parses JSON, validates via `validateAppStore()`
   - Pure logic separated from UI for testability

2. **`src/app/settings/page.tsx`** -- Modifications
   - Add "Data Import & Export" card section below Data Storage
   - Export button calls `exportStoreAsJSON(store)`
   - Import button opens a hidden file input for `.json` files
   - On file select: parse, validate, show confirmation dialog, then load
   - Show success/error messages inline

3. **Tests** -- `src/lib/__tests__/dataImportExport.test.ts`
   - Export generates correct JSON blob with timestamped filename
   - Import of valid file returns validated store
   - Import of invalid JSON returns error
   - Import of non-JSON file returns error
   - Import of partial data gets defaults filled (via validateAppStore)
