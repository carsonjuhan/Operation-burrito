# S-014: Accessibility Audit and Fixes

**Epic:** E-003 (UX Polish and Accessibility)
**Sprint:** 3
**Story Points:** 5
**Status:** In Progress
**Priority:** P1

## Description

Run an accessibility audit and fix all violations to meet WCAG 2.1 AA. The PRD requires keyboard navigation, color contrast, labeled inputs, and minimum touch targets. This story addresses the gaps identified in the UX Design Document (Section 8.2).

## Acceptance Criteria

- [ ] All interactive elements are keyboard-accessible (Tab, Enter, Escape)
- [ ] All form inputs have associated `<label>` elements or `aria-label`
- [ ] Color contrast meets WCAG 2.1 AA (4.5:1 for normal text, 3:1 for large text)
- [ ] Touch targets are at least 44x44px on mobile
- [ ] Modal dialogs trap focus and can be closed with Escape
- [ ] Sidebar drawer is accessible (focus trap on mobile, Escape to close)
- [ ] Icon-only buttons have descriptive `aria-label` attributes
- [ ] Collapsible sections have `aria-expanded` attribute
- [ ] Skip-to-content link is present as first focusable element
- [ ] Focus-visible styles are clear and consistent across all interactive elements

## Tasks

1. **Skip-to-content link** -- Add visually-hidden skip link as first focusable element in layout.tsx
2. **Focus-visible styles** -- Add global `focus-visible` ring styles in globals.css for buttons, links, and interactive elements
3. **Modal focus trapping** -- Add focus trap to Modal.tsx so Tab cannot escape to background content; add `role="dialog"`, `aria-modal="true"`, `aria-labelledby`
4. **Sidebar focus trap** -- Trap focus in mobile sidebar drawer when open; close on Escape
5. **Icon-only button aria-labels** -- Add `aria-label` to: hamburger menu, close buttons, edit/delete/toggle icon buttons across all pages
6. **Collapsible sections aria-expanded** -- Add `aria-expanded` to category group toggle buttons on Items page
7. **Minimum touch targets** -- Ensure 44x44px minimum on: nav items, icon buttons, filter pills, toggle buttons (via `min-h-[44px] min-w-[44px]` or padding adjustments)
8. **Color contrast fixes** -- Upgrade `text-stone-300` informational text to `text-stone-500`; upgrade borderline `text-stone-400` helper text to `text-stone-500` where it conveys meaningful information
9. **aria-live regions** -- Add `aria-live="polite"` to timer display and dynamic feedback areas

## Technical Notes

- Focus trap implementation: use a lightweight approach (track first/last focusable elements, intercept Tab/Shift+Tab)
- Modal already handles Escape key; needs focus trap and ARIA role additions
- Touch target changes should use Tailwind utility classes for responsive sizing
- Color contrast changes should be conservative -- only upgrade where text conveys meaningful information, not decorative elements
- The `prefers-reduced-motion` media query is already respected in globals.css
