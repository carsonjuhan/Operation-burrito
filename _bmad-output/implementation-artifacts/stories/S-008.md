## Description

Detect when local changes and remote Gist changes have diverged since the last sync. Warn the user before a pull would overwrite unsaved local changes, and before a push would overwrite newer remote changes.

Currently, push and pull operations execute without checking whether the other side has been modified since the last sync. This can lead to silent data loss -- a pull overwrites local changes the user made since last sync, or a push overwrites remote changes made from another device.

This story adds a `lastModifiedAt` timestamp that updates on every local mutation, a `fetchGistUpdatedAt()` helper to check the remote timestamp, and confirmation dialogs for both conflict scenarios.

---

## Acceptance Criteria

- [ ] Track a `lastModifiedAt` timestamp that updates on every local mutation
- [ ] On pull: compare local `lastModifiedAt` with `lastSynced`; if local changes exist since last sync, show a confirmation dialog with details
- [ ] On push: fetch Gist `updated_at` and compare with `lastSynced`; if remote is newer, warn that pushing will overwrite remote changes
- [ ] User can choose to proceed or cancel in both cases
- [ ] Timestamps are displayed in a human-readable format
- [ ] Unit tests verify conflict detection logic
- [ ] `npm run build` and `npx vitest run` pass with no regressions

## Technical Context

### Current State
- `src/lib/gistSync.ts` -- push/pull execute without conflict checks
- `src/hooks/useStore.ts` -- `update()` callback persists to localStorage and triggers auto-sync, but does not track a modification timestamp
- `src/app/settings/page.tsx` -- `handlePush()` and `handlePull()` have no pre-flight conflict checks
- `src/types/index.ts` -- `AppStore` has no `lastModifiedAt` field
- `src/lib/gistSync.ts` -- `getLastSynced()` returns the last sync time from localStorage

### Implementation Plan

1. **`src/types/index.ts`** -- Add optional `lastModifiedAt?: string` to `AppStore`

2. **`src/lib/conflictDetection.ts`** -- New module
   - `getLastModifiedAt(): string` -- reads from localStorage
   - `setLastModifiedAt(): void` -- sets current time to localStorage
   - `hasLocalChanges(lastSynced: string): boolean` -- compares lastModifiedAt > lastSynced
   - `hasRemoteChanges(gistUpdatedAt: string, lastSynced: string): boolean` -- compares remote > lastSynced
   - `formatTimestamp(iso: string): string` -- human-readable formatting
   - `ConflictInfo` type -- describes the conflict for UI display

3. **`src/lib/gistSync.ts`** -- Add `fetchGistUpdatedAt(pat, gistId)` function
   - GET the Gist metadata, return `updated_at` field
   - Used by push to check remote freshness before overwriting

4. **`src/hooks/useStore.ts`** -- Modifications
   - Call `setLastModifiedAt()` in the `update()` callback after every mutation

5. **`src/app/settings/page.tsx`** -- Modifications
   - Before pull: check `hasLocalChanges()`, show conflict dialog if true
   - Before push: call `fetchGistUpdatedAt()`, check `hasRemoteChanges()`, show conflict dialog if true
   - Confirmation dialogs with proceed/cancel using project's existing pattern

6. **`src/lib/syncValidation.ts`** -- Handle `lastModifiedAt` field in validation

7. **Tests** -- `src/lib/__tests__/conflictDetection.test.ts`
   - `hasLocalChanges` returns true/false correctly
   - `hasRemoteChanges` returns true/false correctly
   - Edge cases: empty timestamps, equal timestamps
   - `formatTimestamp` produces readable output
