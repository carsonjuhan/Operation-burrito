## Description

Add a web app manifest (`manifest.json`) with app name, icons in multiple sizes, theme colors, and `display: "standalone"` so the app can be installed to the home screen on Android and iOS. Generate simple SVG-based app icons with the burrito theme. Implement a custom install banner using the `beforeinstallprompt` event so users are prompted to install after visiting. Add Apple-specific meta tags in the root layout for iOS PWA support.

---

## Acceptance Criteria

- [ ] `manifest.json` is present in `public/` with name, short_name, icons (192px, 512px), theme_color, background_color, display: "standalone"
- [ ] App icons use the burrito theme in multiple sizes (SVG source, rendered as PNG placeholders)
- [ ] Custom install banner appears using `beforeinstallprompt` event
- [ ] Install banner can be dismissed and does not reappear for that session
- [ ] Installed app opens in standalone mode without browser chrome
- [ ] Apple meta tags are present in layout.tsx for iOS support (apple-mobile-web-app-capable, apple-mobile-web-app-status-bar-style, apple-touch-icon)
- [ ] Manifest is linked in the HTML head via metadata export
- [ ] Unit tests cover the install prompt hook logic
- [ ] `npx vitest run` passes with new tests
- [ ] `npm run build` succeeds

## Technical Context

### Current State
- S-018 implemented the service worker (`public/sw.js`) and registration hook (`src/hooks/useServiceWorker.ts`)
- The app is a static export deployed to GitHub Pages at `/Operation-burrito/`
- `public/` directory contains only `sw.js`
- No manifest or app icons exist yet
- The root layout is in `src/app/layout.tsx`

### Architecture Decisions
- **ADR: SVG icons** -- Use inline SVG to generate burrito-themed icons. These are placed as static SVG files in `public/icons/` for maximum compatibility. No build-time image generation is needed.
- **ADR: beforeinstallprompt hook** -- The install prompt logic is encapsulated in a `useInstallPrompt` hook and rendered by an `InstallBanner` component. The banner auto-hides when the user dismisses it (stored in sessionStorage).
- **ADR: manifest.json in public/** -- The manifest is a static file in `public/` and linked via Next.js metadata in layout.tsx. The `start_url` and icon paths must account for the production `basePath`.

### Relevant Files
- `public/sw.js` -- existing service worker (S-018)
- `src/app/layout.tsx` -- root layout, needs manifest link and Apple meta tags
- `next.config.js` -- basePath configuration

## Implementation Tasks

### Task 1: Create app icons (est. 15 min)

Create SVG icon files in `public/icons/`:
- `icon-192.svg` (192x192)
- `icon-512.svg` (512x512)
- `icon-180.svg` (180x180 for Apple touch icon)
- Simple burrito-themed design using the sage color palette

### Task 2: Create manifest.json (est. 10 min)

Create `public/manifest.json` with:
- `name`: "Operation Burrito"
- `short_name`: "Burrito"
- `description`: "Baby preparation tracker"
- `start_url`: "/Operation-burrito/"
- `display`: "standalone"
- `theme_color`: "#4d6b52" (sage-600)
- `background_color`: "#f4f7f4" (sage-50)
- `icons`: array referencing the icon files

### Task 3: Update layout.tsx with manifest link and Apple meta tags (est. 10 min)

- Add manifest link to metadata export
- Add Apple meta tags: apple-mobile-web-app-capable, apple-mobile-web-app-status-bar-style, apple-touch-icon
- Add theme-color meta tag

### Task 4: Create install prompt hook and banner component (est. 30 min)

- Create `src/hooks/useInstallPrompt.ts` -- captures `beforeinstallprompt` event, exposes prompt trigger and dismiss
- Create `src/components/InstallBanner.tsx` -- renders a dismissible banner with "Install App" button
- Add InstallBanner to layout.tsx

### Task 5: Write tests (est. 20 min)

- Test `useInstallPrompt` hook logic
- Test `InstallBanner` component rendering and dismiss behavior

### Task 6: Verify build and tests (est. 5 min)

- Run `npx vitest run` -- all tests pass
- Run `npm run build` -- static export succeeds with manifest and icons in output
