## Description

Monitor localStorage usage after each write and warn the user when approaching the browser's storage limit (~5 MB). Display current usage in the Settings page and show a warning badge in the sidebar when usage exceeds 80%.

Browsers typically allow ~5 MB of localStorage per origin. Since all app state is persisted to localStorage (and may grow significantly with photos in the future per S-032), proactive monitoring prevents silent data loss when the limit is hit.

---

## Acceptance Criteria

- [ ] A utility function `getLocalStorageSize()` calculates the approximate byte size of all localStorage data for the app's storage key
- [ ] After each store save, the current size is tracked and available via the store context
- [ ] If usage exceeds 80% of the estimated 5 MB limit (~4 MB), a warning badge appears in the sidebar near the Settings link
- [ ] The Settings page displays a "Data Storage" section showing current data size (e.g., "1.2 MB of ~5 MB used") with a visual progress bar
- [ ] When usage is high, the warning suggests enabling GitHub Gist sync as a backup
- [ ] The storage size calculation does not degrade performance (calculated after save, not on every render)
- [ ] Unit tests verify size calculation logic and threshold detection
- [ ] `npm run build` and `npx vitest run` pass with no regressions

## Technical Context

### Current State
- `src/hooks/useStore.ts` contains the `saveStore()` function that writes to localStorage after every mutation
- `src/components/Sidebar.tsx` renders navigation and a sync status footer
- `src/app/settings/page.tsx` has GitHub sync configuration and Amazon registry sections
- Storage key is `"operation-burrito-store"` (defined in `useStore.ts`)

### Implementation Plan

1. **`src/lib/storageMonitor.ts`** -- Utility module
   - `getStorageSize(): { bytes: number; formatted: string }` -- calculates size of the app's localStorage entry
   - `getStoragePercent(): number` -- returns usage as percentage of estimated 5 MB limit
   - `isStorageWarning(): boolean` -- returns true if usage exceeds 80%
   - `STORAGE_LIMIT_BYTES = 5 * 1024 * 1024` constant
   - `WARNING_THRESHOLD = 0.8` constant

2. **`src/hooks/useStore.ts`** -- Modifications
   - After `saveStore()`, calculate and expose `storageSize` state
   - Add `storageSize` and `storagePercent` to the returned hook value

3. **`src/components/Sidebar.tsx`** -- Modifications
   - Import storage monitoring from store context
   - Show an amber warning badge next to Settings link when storage exceeds 80%

4. **`src/app/settings/page.tsx`** -- Modifications
   - Add a "Data Storage" card section between the Amazon Registry section and the Gist ID section
   - Show a progress bar (sage when OK, amber at 60-80%, red above 80%)
   - Display formatted size text (e.g., "1.2 MB of ~5 MB")
   - When above 80%, show a warning message suggesting Gist sync

5. **Tests** -- `src/lib/__tests__/storageMonitor.test.ts`
   - `getStorageSize()` returns correct byte count for known data
   - `getStoragePercent()` returns correct percentage
   - `isStorageWarning()` returns true/false at correct thresholds
   - Handles empty localStorage gracefully
